## Dual 256 grid (two 128s stacked)

- **Serialosc list ports** (e.g. from `serialosc_list_devices`) are **device send ports** (LED + `/sys/…` **to** each grid), not ports you bind with `udpreceive` for keys.
- **Keys** require each device to be told `**/sys/host`**, `**/sys/port**`, and `**/sys/prefix**` (e.g. `/box` to match `p oscreceiveroute`).
- `**grid_composite_2x128**` (`grid_composite_2x128.maxpat` + `grid_composite_2x128.js`):
  - Args `#1` `#2` = the two **device UDP ports** (LED).
  - Local `**udpreceive 58901`** / `**58902**` for keys; on load, `**delay 200**` then send each device `/sys/host 127.0.0.1`, `/sys/port <listen>`, `/sys/prefix /box`.
  - **JS**: 3 inlets (parent LED stream, top keys, bottom keys); outlets 0/1 → two `udpsend`, outlet 2 → parent OSC. `**/monome/…` → `/box/…`** rewrite for LED + passthrough where needed (`swapMonomeToBox`).
- `**_mlr.maxpat**`: `grid_composite_2x128` is **1 inlet / 1 outlet** at the parent level; remove old extra patchlines that assumed 3 monome-style outlets.

## Session — 2026-03-28 — Mute sync (`muted[]`, `output.maxpat`, grid)

**Goal:** Keep `grid_router.js` `**muted[]`** aligned with the real mute path: `**output.maxpat**` mute `**textbutton**` (`obj-6`), `**s #1[pl]mute**`, and `**[box]mute**` (same 0/1 meaning as `pl`).

**Convention (single source of truth)**

- `**muted[i] === 1`** → channel **muted**; `**0`** → **unmuted** (matches `**messnamed(ch + "[box]mute", …)`**).
- **Grid LED (mod page, row 0):** **bright** = unmuted, **off/dim** = muted (`muted[x] ? 0 : 15` in `drawMuteRow`).

**Bug that was fixed**

- `**muteUpdate(ch, val)`** had `**muted[ch-1] = val ? 0 : 1**`, which **inverted** the patch. It must mirror `**[box]mute`**: `**muted[ch-1] = val ? 1 : 0**` (or equivalent), so `**val === 1**` means muted.

**Max wiring**

- `**output.maxpat`:** `**obj-6`** (mute button) → `**prepend muteUpdate #1**` (`obj-173`) → `**s gridrouter**` (`obj-35`), parallel to the existing volume path (`prepend volumeUpdate #1` → `gridrouter`). Each voice sends e.g. `**muteUpdate <#1> <0|1>**` when the UI toggles.

**JS behavior**

- `**anything()`** already routes `**muteUpdate**` to `**muteUpdate()**` (same pattern as `**volumeUpdate**`).
- `**handleModMute**` should call `**drawMuteRow()**` when `**kmod === 2**` after updating `**muted[]**` and `**messnamed**`, so grid LEDs refresh when toggling from the grid (not only when `**muteUpdate**` arrives from the patch).
