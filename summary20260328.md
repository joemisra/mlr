# Session summary вҖ” 2026-03-28 (mlr / grid unification)

Context: unifying drawing, animation, and routing after removing `[p switcher]`.

---

## What was `[p switcher]`?

Old subpatcher in `_mlr.maxpat` that handled:

- **`r mlrpageled`** вҶ’ constrain values (0вҖ“15) вҶ’ `pack 0 0 0` вҶ’ outlet вҶ’ `prepend setcell` вҶ’ `s togridmatrixio` (forwarding playback-head LEDs to the varibright matrix bridge).
- **`r pagechange`** вҶ’ `sel 2` вҶ’ `s box_clear` (clearing grid on kmod change).
- **`r randoggu`** вҶ’ LED animation on col 9 row 0 (old randomize indicator).
- **`r rawpress`** вҶ’ `s box/press_a` (routing raw presses to another bus).

After removal, kmod changes and mod-page drawing were already handled by `grid_router.js`, but the **normal-mode playback-head LED path was broken** because nobody was receiving `s mlrpageled` anymore.

---

## Changes made

### `grid_router.js` вҖ” new LED bridge (replaces `[p switcher]`)

**New message handlers:**

- **`boxled col row level`**: Receives single-cell LED data from `s box/led` (channel playback heads from `[p chnls]`). Only forwards to `setcell` when `kmod === 1` (normal mode). On overlay pages (kmod 2/3/4), playback LEDs are suppressed so the overlay UI isn't overwritten.
- **`boxledrow row lev0 lev1 вҖҰ`**: Full-row LED update from `s box/led_row`.
- **`boxledcol col lev0 lev1 вҖҰ`**: Full-column LED update from `s box/led_col`.

All three call `led(x, y, level)` вҶ’ `outlet(1, "setcell", вҖҰ)` вҶ’ `s togridmatrixio` вҶ’ `grid_matrix_bridge.js`.

**Automation playback guard:**

- Added `playbackDispatching` flag. When `clockTick()` dispatches recorded events back through `dispatch()`, the flag prevents those events from being re-recorded (infinite loop risk).

**Edition-aware grid dimensions:**

- Added `gridWidth` / `gridHeight` variables, updated when `edition` message arrives (64вҶ’8Г—8, 128вҶ’16Г—8, 256вҶ’16Г—16).
- All drawing loops (`ledRow`, `ledCol`, `drawInsertFxBlock`, `drawRandomizeColumn`, `drawOctaveColumns`, `drawReverseColumn`, `animateLeds`) now use `gridWidth`/`gridHeight` instead of hardcoded 16.
- `dispatch()` clamps to `gridWidth-1` / `gridHeight-1`.

### `_mlr.maxpat` вҖ” `[p grid_drawing]` re-routing

The `[p grid_drawing]` subpatcher previously sent LED data to `s mlrpageled` (dead after `[p switcher]` removal). Updated:

- **`s mlrpageled`** вҶ’ renamed to **`s gridrouter`**.
- **`prepend boxled`** inserted between `t b l` and `s gridrouter` for `r box/led`, `r box/test`, `r box/shutdown`, `r box/intensity`.
- **`r box/led_row`** вҶ’ separated from the main path вҶ’ **`prepend boxledrow`** вҶ’ `s gridrouter`.
- **`r box/led_col`** вҶ’ separated from the main path вҶ’ **`prepend boxledcol`** вҶ’ `s gridrouter`.

The animation path (`r rawpress` вҶ’ `switch 2` (kmod-gated) вҶ’ `jstrigger` вҶ’ `pack line вҖҰ` вҶ’ `s togridmatrixanim`) is **unchanged** вҖ” keyframe fade animations still go directly to `grid_anim_engine.js`.

### Deprecated files

- **`routers.js`**: Added deprecation notice. Uses row-major order (row, col, state) which is incompatible with `grid_router.js` (col, row, state). All functionality now in `grid_router.js`.
- **`groupsproc.js`**: Added deprecation notice. Had a bug where `muted` (array) was treated as a scalar. Functionality now in `grid_router.js` (`handleRow0Channel`, `handleModMute`).

## LED output paths (single source of truth)

| Source | Message bus | Handler | Matrix path |
|--------|-----------|---------|-------------|
| Channel playback heads | `s box/led` | `grid_router.js` `boxled()` | вҶ’ `setcell` вҶ’ `grid_matrix_bridge.js` |
| Channel row clears | `s box/led_row` | `grid_router.js` `boxledrow()` | вҶ’ `setcell` вҶ’ `grid_matrix_bridge.js` |
| Mod page UI | (internal) | `grid_router.js` `led()` | вҶ’ `setcell` вҶ’ `grid_matrix_bridge.js` |
| Button press fades | `s togridmatrixanim` | `grid_anim_engine.js` `line()` | вҶ’ writes to jit.matrix directly |
| Pattern replay | `prepend handleOldPatternOut` | `grid_router.js` `handleOldPatternOut()` | вҶ’ `led()` вҶ’ `setcell` |

All LED paths now converge through `grid_matrix_bridge.js` вҶ’ `/grid/led/level/map` вҶ’ serialosc. No more dual `box/led` vs `grid_matrix_bridge` conflict.
